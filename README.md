# Desafios Maratona Java

Esta pasta foi criada para fixar o que voce ja estudou em `maratona-java`.

## Como estudar

Use este ciclo em cada desafio:

1. Leia o enunciado antes de olhar o codigo.
2. Explique em voz alta qual problema voce precisa resolver.
3. Escreva a solucao sem pesquisar por 20 a 30 minutos.
4. Rode o programa e compare com a saida esperada.
5. Anote o que travou em `99-revisoes/diario-de-erros.md`.
6. Refaca o mesmo desafio no dia seguinte sem olhar a primeira solucao.

Esse metodo mistura pratica ativa, repeticao espacada e revisao dos proprios erros. A ideia nao e decorar sintaxe solta, e fazer seu cerebro recuperar o caminho da solucao varias vezes.

## Ordem recomendada

1. `01-fundamentos-condicionais`
2. `02-repeticao-busca`
3. `03-arrays`
4. `04-arrays-multidimensionais`
5. `05-classes-objetos`
6. `06-metodos`

## Regra importante

Nao apague os comentarios dos desafios. Use eles como checklist. Quando terminar uma etapa, marque com `OK` do lado do comentario.

Exemplo:

```java
// TODO 1: criar uma variavel idade. OK
```

## Como rodar

Entre na pasta do desafio e rode:

```bash
javac NomeDoArquivo.java
java NomeDoArquivo
```

Quando houver dois arquivos no mesmo desafio, compile todos:

```bash
javac *.java
java NomeDoTeste
```
